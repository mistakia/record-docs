## Project Overview

Static documentation site for the **Record** distributed audio file system. The site is generated by `build.js` from the templates in `src/` and the specification chapters in `spec/`. Output goes to `dist/`.

## Layout

```
spec/                                # The protocol specification (canonical)
  1-overview.md
  2-data-model.md
  3-identity-and-access.md
  4-library-structure.md
  5-network-protocol.md
  6-content-processing.md
  7-http-api.yaml                    # OpenAPI for the HTTP API draft
  fixtures/
src/                                 # Site templates
  index.jade
  index.js
  index.styl
  readme.js
build.js                             # Build script
dist/                                # Output
```

## Build

```bash
yarn install
node build.js                        # Build the static site to dist/
```

## Editing the Spec

The numbered `spec/*.md` chapters are the protocol's canonical text. When editing:

- Keep the section order stable; consumers link by anchor.
- Coordinate changes that affect on-wire behavior with `record-node`.
- Update `7-http-api.yaml` together with any HTTP API change.
